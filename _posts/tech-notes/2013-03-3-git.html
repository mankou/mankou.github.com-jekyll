1 简介

  1. git是一个分布式版本控制系统。最初是用于管理linux内核代码的，后来变得很流行。

2 使用

要在windows下使用git总体上是命令行工具+图形化插件. 命令行工作有两种:msysgit 和cygwin
图形化的插件有:TortoiseGit 和git extensions
msysgit和TortoiseGit组合 这是最常用的不用说
msysgit和git extensions的组合,也还可以,而且安装git extensions时会帮你安装msysgit的
cygwin+git extensions+Git Source Control Provider: 是网上推荐的,我没有用成功. 配置好后,可以打开库,但不能clone push.

2.1 msysgit

windows下的git客户端软件,我不知道这算不算客户端软件。由于git是分布式的，每个终端都能独立进行版本控制。你不要远程服务器也能进行版本控制。所以这里也不知道把它叫客户端对不对

2.2 TortoiseGit

基于 msysgit 的 GUI 程序

2.3 EGit

Eclipse下用于git操作的插件

2.4 github

拖管git代码的网站

2.5 git-vim的使用

vim的git插件，在vim下利用git进行版本控制

  5. 文件内容乱码问题
    如果本地文件中有中文内容，一般其编码格式为chinese,但git的工具一般都认utf-8编码.所以会产生乱码.例如:
    1. 用vim编写的c语言,因为要用gcc编译其编码格式用chinese,但导入github后网页上显示的是乱码. 但如果改成utf-8编码,则运行生成的exe 又是乱码.
    2. 批处理文件一般文件编码格式也是chinese,但导入github网页上网页上显示的是乱码.用git Git Extensions查看其内容也是乱码.
      这两个问题实质上是一个问题,即windows的cmd窗口不支持utf-8
  6. git文件名乱码
    如果文件名是中文，则push到远程再clone下来时可能会出现文件名乱码.
    2012年7月31日发现由git控制的代码库都出现不同程度的中文文件名乱码问题. 包括 bat c语言库 java库 GTD库. 而且这些乱码文件你既删不了,也不能提交,因为人家说你路径不对,找不到文件.当时并没有解决,基本上是丢弃了原来的版本控制信息.
    也有可能是你升级git版本的原因,以前提交那么多次如果有乱码应该就发现了.正好这几天升级了git的版本
